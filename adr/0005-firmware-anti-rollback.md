# ADR-0005: 펌웨어를 되돌리기 불가능하게 3중으로 보장한다

- **상태(Status):** Accepted
- **일자(Date):** 2026-02-28
- **작성자(Author):** LEE SEUNG JU
- **관련 레포:** ICONIA-HW
- **태그:** 임베디드 · 보안 · 펌웨어 공급망

## 맥락 (Context)

ICONIA는 아이 곁에 두는 **물리 기기(인형)** 입니다. 펌웨어 보안이 뚫리면 물리적 회수 없이는 복구가 어렵습니다.

- **강제 다운그레이드:** 공격자가 취약점이 있던 **옛 펌웨어로 되돌리면**, 이미 패치한 구멍을 다시 열 수 있습니다.
- **펌웨어 위조:** 서명되지 않은 이미지가 부팅되면 기기를 통째로 장악당합니다.
- **규모:** 한 대가 아니라 **전체 기기군(fleet)** 이 동시에 위험해집니다.

무선 업데이트(OTA)는 편의를 주지만, 동시에 **원격 공격 표면**이기도 합니다.

## 결정 (Decision)

“되돌릴 수 없고, 정품만 부팅된다”를 **하드웨어까지 내려가는 3중 방어**로 보장한다.

1. **안티-롤백 3중 버전 잠금**
   - `semver` (애플리케이션 레벨 버전)
   - `firmware secure_version` (펌웨어 메타 레벨)
   - **eFuse `SECURE_VERSION`** — 칩에 물리적으로 새겨져 **되돌릴 수 없는** 최소 허용 버전.
   → 셋 중 하나라도 과거 버전이면 부팅/설치를 거부.
2. **Secure Boot V2 (ECDSA 서명 검증)** — 부팅 시 **서명이 검증된 정품 이미지만** 실행.
3. **TLS 1.2 + SPKI 피닝** — OTA/명령 채널은 **미리 지정한 인증서만 신뢰**해 중간자 공격을 차단하고, 응답 헤더 기반 명령 채널로 업데이트·프로비저닝을 전달.

## 고려한 대안 (Alternatives)

| 대안 | 장점 | 채택하지 않은 이유 |
|---|---|---|
| 소프트웨어 버전 체크만 | 구현 쉬움 | 펌웨어 조작으로 우회 가능(하드웨어 앵커 없음) |
| Secure Boot만 | 위조 차단 | 서명된 과거 취약 버전으로의 다운그레이드는 못 막음 |
| eFuse까지 3중(채택) | 위조·다운그레이드 모두 원천 차단 | eFuse는 **비가역** → 버전 운용 실수 시 복구 불가(감수) |

## 결과 (Consequences)

**긍정적**
- 옛 취약 펌웨어로의 **강제 다운그레이드가 물리적으로 불가능**해진다.
- **정품 서명 이미지만 부팅**되어 위조 펌웨어를 원천 차단.
- OTA 채널이 피닝으로 보호되어 **중간자·가짜 서버 공격을 차단.**

**부정적 / 감수한 비용**
- **eFuse는 되돌릴 수 없다.** 버전 번호를 잘못 올리면 **해당 기기는 롤백 불가** → 릴리즈 프로세스에 강한 규율이 필요.
- 인증서 피닝은 **인증서 교체 시 OTA로 신중히 갱신**하지 않으면 기기가 서버와 단절될 위험.
- 서명·검증·버전 관리로 **빌드/릴리즈 파이프라인이 복잡**해진다.

**후속 조치**
- eFuse 버전 증가를 **릴리즈 체크리스트의 명시적 승인 항목**으로 고정.
- 인증서 로테이션 시 “구·신 인증서 동시 신뢰 창” 절차를 별도 ADR로 문서화.
