# ADR-0003: 인증을 토큰 회전 + 재사용 감지로 탈취에 강하게 만든다

- **상태(Status):** Accepted
- **일자(Date):** 2026-02-20
- **작성자(Author):** LEE SEUNG JU
- **관련 레포:** ICONIA-SERVER
- **태그:** 보안 · 인증

## 맥락 (Context)

ICONIA는 아이의 대화·미디어 등 **민감 데이터**를 다룹니다. 로그인 유지를 위한 refresh 토큰이 탈취되면 문제가 큽니다.

- refresh 토큰은 수명이 길어, **한 번 새어나가면 장기간 악용**됩니다.
- 단순 만료 방식만으로는 탈취를 **탐지할 수 없습니다.** 정상 사용자와 공격자가 같은 토큰을 쓰는지 구분이 안 됩니다.
- 운영자 계정은 사용자 데이터에 광범위하게 접근하므로 **일반 인증보다 강한 보호**가 필요합니다.

## 결정 (Decision)

1. **JWT RS256 + Access/Refresh 회전(rotation)** — refresh를 쓸 때마다 **새 토큰 쌍을 발급**하고 이전 refresh는 폐기한다.
2. **재사용 감지(family revoke)** — 이미 폐기된(회전된) refresh가 다시 들어오면, 그 **토큰 계열(family) 전체를 즉시 무효화**한다. → 탈취범과 정상 사용자 중 하나가 반드시 옛 토큰을 쓰게 되므로, **탈취가 자동으로 드러나고 봉쇄**된다.
3. **운영자 2차 인증(TOTP MFA)** — ADMIN 로그인에 시간 기반 OTP를 추가한다.
4. **감사로그 해시 사슬(hash-linked chain)** — 각 로그 레코드가 이전 레코드의 해시를 포함해, **한 줄만 위·변조해도 사슬이 깨져 탐지**된다.

## 고려한 대안 (Alternatives)

| 대안 | 장점 | 채택하지 않은 이유 |
|---|---|---|
| 장수명 refresh + 만료만 | 단순 | 탈취 탐지 불가, 피해 장기화 |
| 세션 서버(stateful) | 즉시 무효화 쉬움 | 분산 확장성↓, SSOT 원칙과 상충 |
| 토큰 회전 + family revoke(채택) | 탈취 자동 탐지·봉쇄 | 토큰 관리·클라이언트 재발급 처리 복잡도↑(감수) |

## 결과 (Consequences)

**긍정적**
- 토큰이 탈취돼도 **다음 회전 시점에 자동으로 발각·봉쇄**된다. 피해 창이 극적으로 짧아진다.
- 감사로그가 **위·변조 시 즉시 탐지**되어 사후 조사 신뢰도가 높다.
- 운영자 계정 탈취 난이도가 MFA로 크게 올라간다.

**부정적 / 감수한 비용**
- 회전 로직 때문에 **클라이언트가 “재발급 경합/네트워크 단절” 상황을 정교하게 처리**해야 한다(잘못하면 정상 사용자가 로그아웃됨).
- family revoke는 **오탐 시 정상 사용자를 끊을 수 있어**, 다중 기기 시나리오를 신중히 설계해야 한다.
- TOTP는 운영자에게 **약간의 사용 마찰**을 준다.

**후속 조치**
- 회전 경합으로 인한 강제 로그아웃률을 지표화.
- 다중 기기(family 분리) 정책을 별도 문서로 정리.
